# Objetivo

Definir toda a engenharia de firmware do Project SIGMA.

---

# Filosofia

Firmware industrial deve ser:

Determinístico

Modular

Escalável

Seguro

Documentado

Testável

Nunca escrever código apenas para funcionar.

---

# Arquitetura

HAL

↓

Drivers

↓

Storage

↓

Services

↓

Application

Nunca violar esta hierarquia.

---

# Princípios

Responsabilidade única.

Baixo acoplamento.

Alta coesão.

Sem alocação dinâmica.

Sem exceções C++.

Sem RTTI.

Sem String.

---

# Memória

Nunca utilizar:

new

delete

malloc

free

std::vector

std::list

std::string

Sempre utilizar buffers estáticos.

---

# Tempo Real

Evitar bloqueios.

Evitar delay().

Sempre utilizar millis().

Preparar para FreeRTOS.

---

# Watchdog

Toda tarefa crítica deve alimentar o WDT.

Nunca desabilitar watchdog em produção.

---

# Código

Funções pequenas.

Classes pequenas.

Arquivos especializados.

Comentários conforme CCS.

Interface Serial conforme SUS.

Arquitetura conforme RAM.

---

# Logs

Separar:

Debug

Info

Warning

Error

Nunca imprimir continuamente em produção.

---

# Objetivo Final

Construir um firmware industrial robusto, determinístico e preparado para evolução contínua.
