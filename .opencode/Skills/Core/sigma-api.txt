# SIGMA API Skill

# Objetivo

Esta skill define toda a arquitetura da API do SIGMA Studio.

A API representa o contrato oficial entre todos os componentes do ecossistema SIGMA.

Toda comunicação entre sistemas deve ocorrer através de contratos bem definidos, previsíveis e versionados.

---

# Princípios

Toda API deve ser:

Consistente

Versionada

Tipada

Documentada

Escalável

Previsível

Nunca quebrar compatibilidade sem necessidade.

---

# Consumidores da API

A API deverá atender:

SIGMA Studio

Firmware SIGMA

Dashboard Web

Aplicativo Mobile (futuro)

Integrações externas

Scripts de automação

Nunca desenvolver endpoints exclusivos para um cliente.

Sempre pensar na reutilização.

# Versionamento

Toda rota deve iniciar por:

/api/v1/

Nunca publicar endpoints sem versionamento.

Quando houver mudanças incompatíveis:

Criar:

/api/v2/

Nunca modificar silenciosamente uma versão existente.

---

# JSON

Sempre utilizar:

camelCase

Nunca misturar snake_case.

---

# Datas

Utilizar ISO-8601.

Exemplo

2026-07-05T15:30:00Z

Nunca enviar datas em formato local.

---

# Timestamp

Sempre em UTC.

Converter apenas na interface.

---

# Identificação

Todo dispositivo possui:

deviceId

serialNumber

firmwareVersion

hardwareRevision

model

Nunca identificar dispositivos apenas pelo nome.

---

# Respostas

Sempre retornar:

success

data

message

timestamp

requestId

Exemplo

{
  "success": true,
  "data": {},
  "message": "Operação realizada.",
  "timestamp": "...",
  "requestId": "..."
}

---

# Erros

Mesmo padrão para todos.

{
  "success": false,
  "error": {
      "code": "...",
      "message": "...",
      "details": []
  },
  "timestamp": "...",
  "requestId": "..."
}

Nunca retornar exceções internas.
