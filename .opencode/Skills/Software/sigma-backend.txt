# SIGMA Backend Skill

# Objetivo

Esta skill é responsável por toda a arquitetura Backend do SIGMA Studio.

O Backend representa o núcleo da plataforma.

Sua missão é garantir um sistema modular, escalável, seguro e preparado para milhares de dispositivos conectados.

# API

Toda comunicação externa deve passar pela API.

Nunca acessar diretamente os Services.

---

# Domain

Representa as regras de negócio.

Nunca acessar banco.

Nunca acessar MQTT.

Nunca acessar FastAPI.

---

# Services

Responsáveis por orquestrar:

- dispositivos
- analytics
- alertas
- processamento
- usuários

---

# Repositories

Responsáveis exclusivamente pelo acesso aos dados.

Nunca conter regras de negócio.

---

# Models

Representam as entidades do banco.

---

# Schemas

Responsáveis pela validação Pydantic.

Nunca reutilizar Models SQLAlchemy como Schema.

# MQTT

Todo dispositivo deve possuir:

Device ID

Firmware Version

Timestamp

Status

Health Score

Temperatura

Vibração

Horímetro

Alertas

Nunca enviar mensagens sem identificação.

---

# WebSocket

Responsável por:

Atualizações em tempo real.

Nunca realizar processamento pesado dentro do WebSocket.

---

# Banco

PostgreSQL.

Nunca acessar SQL diretamente.

Sempre utilizar Repository Pattern.

---

# Analytics

Toda análise deve ocorrer em Services especializados.

Nunca misturar Analytics com API.

---

# IA

Criar módulo separado:

ai/

Nunca misturar IA com Telemetria.

---

# Segurança

JWT

Refresh Token

RBAC

HTTPS

Rate Limit

CORS

Nunca armazenar senhas em texto.

Sempre utilizar Argon2.

---

# Logs

Utilizar logging estruturado.

Separar:

INFO

WARNING

ERROR

CRITICAL

Nunca utilizar print().

---

# Configuração

Utilizar:

.env

Pydantic Settings

Nunca utilizar constantes espalhadas.

---

# Dependências

Utilizar Dependency Injection do FastAPI.

Nunca criar singletons manuais.
